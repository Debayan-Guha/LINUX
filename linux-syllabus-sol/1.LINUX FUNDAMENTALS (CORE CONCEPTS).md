# 1.3 Filesystem Hierarchy Standard (FHS)

## 1.3.1 `/` (root directory) purpose
- top level directory
- starting point for the entire Linux filesystem hierarchy.

## 1.3.2 `/bin` vs `/sbin` vs `/usr/bin`

- `/bin` Essential commands for all users (ls, cp, mv).
- `/sbin` Essential system administration commands (fdisk, ip, mkfs).
- `/usr/bin` When you install a new app—like a web browser, a text editor, or a game—its startup command is placed inside /usr/bin so that any user on the computer can open it. Onlyexecutable file (the single file that actually starts the program) is placed in `/usr/bin.`

## 1.3.3 `/etc` (configuration files)
contains the global configuration files for virtually all the applications and services installed on your Linux system

## 1.3.4 `/var` (variable data: logs, spool)
Stores frequently changing data such as logs, mail, caches, databases, and spool files.

## 1.3.5 `/tmp` (temporary files)
- Any user and any app on the computer has permission to create, read, and write files here. It is like a public bulletin board where anyone can pin a note.
- Every single time you restart or turn off your Linux computer, the system completely wipes this folder clean. When the computer boots back up, /tmp is 100% empty.
- Apps use this folder to store background data they need right now, but won't need later.


## 1.3.6 `/home` vs `/root`
- `/home` → Home directories of normal users.
- `/root` → Home directory of the root (administrator) user.


## 1.3.7 `/proc` and `/sys` (virtual filesystems)
- `/proc` A live tracking folder for the CPU (Processor), Memory (RAM), and Running Programs (Processes). Every running program gets its own folder named after its Process ID number `(like /proc/1234)`. If you open the file /proc/meminfo, it shows exactly how much RAM your computer is using this very second.
- `/sys`  A live interface for your physical hardware devices, drivers, and settings (like your screen, battery, USB ports, and keyboard). It tells you what hardware is plugged in, and lets you change how that hardware behaves


## 1.3.8 `/mnt` vs `/media` (mount points)
- `/mnt` A folder reserved for the System Administrator to connect drives manually using commands. A temporary workspace where you manually force a drive to connect when doing system maintenance or recovery.
- `/media` A folder managed entirely by the operating system for plug-and-play devices. Where your USB thumb drives, external SSDs, and CDs automatically appear when you plug them in. You plug in a USB drive named "Backup". Linux automatically creates a folder called /media/Backup and opens it for you. When you unplug the USB, Linux deletes that folder automatically.


## 1.3.9 `/opt` (optional software)
Used for third-party or optional software installed outside the normal package structure. If an app comes from the official Linux App Store (the package manager), its pieces get split up across the system. If you download a massive app directly from a company's website (like Google or Zoom), it usually drops itself entirely into `/opt`.
