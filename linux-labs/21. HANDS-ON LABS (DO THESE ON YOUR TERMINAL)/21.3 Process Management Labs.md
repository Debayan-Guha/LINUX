## 21.3.1.1 Run `sleep 300` in the background
<img width="515" height="287" alt="image" src="https://github.com/user-attachments/assets/726e77b7-c6d7-4f42-be5e-70fa80cf3226" />

```bash
# Run sleep 300 in the background using & . & - Runs command in background
sleep 300 &

# View the background job
jobs

# List all background jobs number and its corresponding PID side-by-side.
jobs -l

# Get PID for a specific job (replace 1 with your specific job number).
jobs -p %1

# Bring all job to foreground
fg

# # Bring particular job to foreground (replace 1 with your specific job number).
fg %1

# terminate the background job (replace 1 with your specific job number).
kill %1
```

## 21.3.1.9 Use `ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%cpu` to see CPU hogs

This command shows processes sorted by CPU usage (highest first) - perfect for finding what's consuming your CPU!

<img width="1366" height="768" alt="image" src="https://github.com/user-attachments/assets/caae6a0b-b448-4c07-8fc7-0ca138ef2ee8" />

BreakDown 
* **`ps`**: Reports a snapshot of the current processes.
* **`-e`**: Selects all processes running on the system.
* **`-o`**: Specifies a user-defined custom output format.
* **`pid,ppid,cmd,%mem,%cpu`**: Defines the columns to display:
  * `pid`: Process ID
  * `ppid`: Parent Process ID
  * `cmd`: Executable name/command
  * `%mem`: Percentage of physical memory used
  * `%cpu`: Percentage of CPU time used
* **`--sort=-%cpu`**: Sorts the results by CPU usage in descending order (highest first due to the `-` prefix).


## 21.3.1.10 Observe process state (R, S, D, Z) using `ps`
Process states show what a process is currently doing. Think of them like status indicators for each running program.

<img width="1366" height="768" alt="image" src="https://github.com/user-attachments/assets/5bb6914f-cf70-422e-8a87-22f8b08f2ade" />

**Linux Process States Explained**

When monitoring processes with tools like `ps` or `top`, the kernel reports the execution status of each process using a specific state symbol.

**Process State Table**

| State | Symbol | Meaning | Example |
| :---: | :---: | :--- | :--- |
| **R** | Running | Process is actively using CPU | `sleep 1` (briefly) |
| **S** | Sleeping | Waiting for something (I/O, event) | `sleep 300`, `ssh`, `bash` |
| **D** | Uninterruptible Sleep | Waiting for disk I/O (cannot be killed) | Reading/writing to a slow disk |
| **Z** | Zombie | Process finished, but parent has not cleaned up | Defunct processes |
| **T** | Stopped | Paused by a system signal | Job paused via `Ctrl+Z` |
| **I** | Idle | Kernel thread sleeping to save power | `kthreadd` |
| **X** | Dead | Process is dying and about to be cleaned up | Rare transient state |
