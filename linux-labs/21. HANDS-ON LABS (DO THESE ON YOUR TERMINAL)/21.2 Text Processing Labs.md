**📊 Flag**

- ***The `-o` Option (Data Isolation) :*** Standard `grep` prints the entire string buffer containing the match. Using `-o` forces the regex engine to truncate everything except the explicit match bytes.
  * **Standard Output:** Prints the full line containing metadata and text.
  * **`-o` Output:** Prints *only* the specific text block captured by the regex pattern. Use When you need clean, isolated data strings instead of the entire text line.

- ***The `-E` Option (Syntax Switching) :*** By default, `grep` parses strings using Basic Regular Expressions (BRE), which treats symbols like `{}` and `()` as literal characters. 
  * Without `-E`, you must escape everything: `\([0-9]\{1,3\}\)`
  * With `-E`, the engine interprets modern regex syntax directly: `([0-9]{1,3})`. Use When your regex pattern utilizes structural operators like `{}` or `()`.

# 21.2.2 sed Lab

## 21.2.2.1 Create `config.txt` with `port=8080` and change it to `port=9090` using sed (dry run first)
<img width="1240" height="241" alt="image" src="https://github.com/user-attachments/assets/26bdbb76-baaa-4c4a-9f4a-278e72c0f544" />

```bash
# 1. Create the config file
echo "port=8080" > config.txt

# 2. Dry run (Prints the change to screen, does NOT modify the file)
sed 's/port=8080/port=9090/' config.txt
```

## 21.2.2.2 Edit `config.txt` in-place to change the port
<img width="1099" height="191" alt="image" src="https://github.com/user-attachments/assets/e82dc935-0572-46d1-b8e6-799614300671" />

```bash
# 1. Real run (Modifies the file in-place)
sed -i 's/port=8080/port=9090/' config.txt
```
- ***With `-i` :*** It intercepts the output and writes the changes directly back into the original file. It overwrites the file on disk.
